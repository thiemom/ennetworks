A brief history of the minimalist web game

The game was inspired by looking at the example included with the csatonce app. Flying from Zurich to Amsterdam, I entertained myself by creating the processing.js script that makes up the game. You can find the script at the bottom of the page.

The game uses canvas to draw on. The ball position and movement is calculated with linear motion equations for x and y.



Forecasting the ball's path is done by taking the gradient (the angle) of the ball position, which is equal to the quotient of the y and x velocities. 


When the left or right boundary are reached, the direction of the ball is reversed. I forecast the path by piecewise calculation of the coordinates where either the left or right boundary are reached. When they are reached the point is taken as start for the next segment and the x velocity is reversed. This is repeated until finally the upper or lower boundary is reached. I put this in a for loop, to avoid any risk of getting stuck in an endless loop.

By the way, I created the images with the equations using an online LaTeX editor.

Some "gotchas" I figured out on the way:

  1. I needed to include background() inside the draw() function to clear the (entire) canvas.
  2. Variable types can change once declared, which is why I sometimes used an explicit typecast.
  3. The processing code keeps running even when the window lost focus. To change this behavior I needed to include the directive /* @pjs pauseOnBlur="true"; */ 
  4. As total newbie I had to figure out the hex color codes. It turned out to be a 6 digit hexadecimal number representing red, green and blue intensity. So, for example bright red is #FF0000, which means 255 intensity red, 00 intensity green and 00 blue. Grey would be equal intensity of red, green and blue (0 < intensity < 255).
